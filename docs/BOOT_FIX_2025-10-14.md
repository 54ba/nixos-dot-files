# NixOS Boot and Environment Variable Fix - Complete

## Date: October 14, 2025

## Issues Identified and Fixed

### 1. **Critical: Read-Only /nix/store (FIXED ✅)**
**Problem**: The /nix/store was mounted as read-only, preventing any system rebuilds.
```bash
# Before:
/dev/sda1 on /nix/store type ext4 (ro,relatime)

# After:
/dev/sda1 on /nix/store type ext4 (rw,relatime)
```

**Solution**: Remounted /nix/store as read-write:
```bash
sudo mount -o remount,rw /nix/store
```

### 2. **Critical: LD_LIBRARY_PATH Configuration Conflict (FIXED ✅)**
**Problem**: Multiple modules were defining `LD_LIBRARY_PATH` uniquely:
- `ai-services.nix` - CUDA libraries
- `pipewire.nix` - PipeWire JACK libraries  
- `configuration.nix` - System libraries

**Error**:
```
error: The option `environment.sessionVariables.LD_LIBRARY_PATH' is defined multiple times while it's expected to be unique.
```

**Solution**: Changed configuration in `/etc/nixos/configuration.nix` line 752:
```nix
# Before (WRONG - caused conflict):
LD_LIBRARY_PATH = lib.mkAfter "/run/current-system/sw/lib";

# After (CORRECT - uses list with lib.mkBefore):
LD_LIBRARY_PATH = lib.mkBefore [ "/run/current-system/sw/lib" ];
```

This allows proper merging with other module definitions.

### 3. **Non-Critical: ACPI BIOS Errors (ACKNOWLEDGED ⚠️)**
**Problem**: Hardware-specific ACPI errors from ThinkPad BIOS:

- ThinkPad ACPI EC errors

**Status**: These are BIOS bugs and don't affect system functionality. They appear in all boots and are safe to ignore.
